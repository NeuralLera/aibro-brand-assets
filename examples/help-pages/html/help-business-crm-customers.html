Мини‑CRM клиентов

4 мин чтения обновлено 14.05.2026 автор: команда СММ АЙБРО

Что это

Мини‑CRM — это карточки клиентов вашего бизнеса с историей покупок, тегами и заметками. Каждая карточка создаётся автоматически при первом контакте клиента с ИИ‑ассистентом в MAX — ничего настраивать не нужно.

Это адресная книга вашего бизнеса: вы видите, кто, что и когда у вас покупал, кого пора пригласить обратно, а кому стоит сделать VIP‑предложение. Подходит пекарням, салонам красоты, мастерским, цветочным, фермерам — любому бизнесу с повторными клиентами.

Как это работает

  1. Клиент впервые пишет ассистенту в MAX — создаётся карточка с именем из MAX‑профиля. Телефон добавляется автоматически, когда клиент его указал в заказе или записи.
  2. Каждый заказ и каждая запись связываются с карточкой — у вас на глазах копится история взаимодействий.
  3. Lifetime Value (LTV) и средний чек считаются автоматически по завершённым заказам.
  4. Теги Новый / Постоянный / Спящий присваиваются авто‑правилами; вы можете добавлять свои собственные теги вручную.

Что внутри карточки

  • Контакт — имя, телефон, email (если клиент указал), MAX‑аккаунт.
  • Статистика — всего заказов, общий LTV, средний чек, дата последней активности.
  • История заказов — что, когда, на какую сумму, в каком статусе.
  • История записей — на какую услугу, к какому мастеру, когда (для салонов и услуг).
  • Заметки от владельца — свободный текст: предпочтения, аллергии, день рождения.
  • Теги — короткие метки для сегментации и фильтрации.
  • Журнал общения — сообщения клиента в MAX и ответы ИИ‑ассистента.
  • Контролы 152‑ФЗ — «Удалить данные клиента» и «Экспорт PDF».

Изоляция клиентов между бизнесами

Если один и тот же человек (например, Мария) сделал заказ у вас и у другого бизнеса на платформе — у каждого бизнеса появится своя отдельная карточка клиента. Это сделано сознательно:

  • ваши теги, заметки и статистика принадлежат только вам;
  • другие владельцы не видят ваших клиентов, а вы — их;
  • даже сама платформа СММ АЙБРО не объединяет клиентов между бизнесами автоматически.

Один человек = разные клиенты у разных бизнесов. Это не баг — это безопасность и соответствие 152‑ФЗ.

Пример карточки клиента

Так выглядит карточка постоянного клиента, которую ИИ‑ассистент собрал из истории заказов:

Клиент #084 · VIP 
{
  "id":    84,
  "name":  "Мария",
  "phone": "+7 921 ... ..",
  "first_seen": "2026‑01‑12",
  "last_seen":  "2026‑06‑10",
  "stats": {
    "orders_total": 14,
    "ltv":          18260,
    "avg_check":    1304
  },
  "tags": [
    "VIP", "постоянный", "любит медовик"
  ],
  "notes": "День рождения 14 марта. Без орехов.",
  "last_order": {
    "id": 142, "total": 1240, "status": "выполнен"
  }
  // данные изолированы — видны только вашему бизнесу
}

Тегирование (3 типа)

  • Системные теги (авто). «Новый» (1 заказ), «Постоянный» (3+ заказов за 90 дней), «Спящий» (0 заказов 30+ дней). Платформа считает их сама.
  • Custom‑теги (от владельца). «VIP», «Корпоратив», «Аллергик», «Любит доставку», «Нужен счёт» — что нужно вашему бизнесу.
  • Smart‑теги (ИИ). «Любит сладкое», «Часто переносит запись», «Берёт большой чек по пятницам» — ИИ выявляет паттерны из истории.

Примеры использования тегов:

  • Запустить рассылку только тегу «VIP» при появлении эксклюзивной услуги.
  • Найти всех с тегом «торт на заказ», чтобы предложить новинку.
  • Отделить теги для разных целей: служебные («ждёт перезвон»), маркетинговые («лояльный»), бизнесовые («b2b»).

Lifetime Value (LTV)

LTV — общая сумма всех оплаченных заказов клиента за всё время взаимодействия с вашим бизнесом. Помогает понять, кто действительно ценный клиент:

  • топ‑10% клиентов по LTV обычно дают 50–70% выручки;
  • по LTV удобно выбирать клиентов для VIP‑программы и эксклюзивных предложений;
  • падение среднего LTV — сигнал, что клиенты стали покупать меньше или ушли.

В этом релизе LTV пересчитывается при смене статуса заказа на «выполнен». Отменённые заказы в LTV не учитываются.

Соответствие 152‑ФЗ

Карточка клиента содержит персональные данные (телефон, имя) — по закону «О персональных данных» (152‑ФЗ) мы обрабатываем их как processor от вашего имени как controller. Что это значит на практике:

  • Маскированные номера в диалогах. В ответах бота и логах ИИ‑ассистента телефон отображается только хвостом (например: «...456»). Полностью — только владельцу и сотрудникам с доступом к CRM.
  • Удаление по запросу. Если клиент попросил удалить данные — в карточке есть кнопка «Удалить данные клиента». Удаляется ПДн (имя, телефон, заметки), история заказов остаётся обезличенной для бухгалтерии.
  • Экспорт PDF. Для ответов на запросы клиентов по 152‑ФЗ — одна кнопка, готовый документ со всеми данными и историей.
  • Договор обработки данных (DPA). Оформляется автоматически при подписке на тариф — отдельных действий от вас не требуется.

Безопасность по умолчанию. Доступ к карточкам строго ограничен по идентификатору бизнеса. Никакой запрос вида «покажи всех клиентов владельца» не вернёт клиентов чужого бизнеса — это исключено на уровне базы данных.

Заметки

Заметки — место для всего, что не помещается в теги:

  • день рождения и предпочтения по подарку;
  • аллергии и противопоказания (для медцентров, кафе, салонов);
  • особенности доставки (домофон не работает, оставлять у консьержа);
  • история разговоров и обещаний клиенту.

Заметки видны только владельцу и сотрудникам с доступом к CRM — клиент их не видит.

Чего пока нет

  • Импорт из CSV/Excel. Появится после релиза. Сейчас клиенты добавляются только через диалог в MAX или вручную через кабинет.
  • Автоматическое слияние дублей. Если у одного клиента два телефона — карточки придётся объединить вручную.
  • Двухсторонняя синхронизация с amoCRM / Bitrix24. Не планируется в ближайших релизах — мини‑CRM создавалась как самодостаточная для малого бизнеса.
  • Встроенный сегментатор. Сегментируем пока только тегами — конструктор условий и воронок появится позже.

Частые вопросы

Дедупликация идёт по телефону, а не по MAX‑аккаунту. Когда клиент укажет тот же номер в заказе, его новое сообщение свяжется со старой карточкой — история не потеряется. До этого момента создастся временная карточка только с именем из MAX, и она автоматически объединится после первого заказа с известным телефоном.
В этом релизе — вручную: открываете карточку‑дубль, нажимаете «Объединить с…» и выбираете основную карточку. История заказов и записей сольётся, теги объединятся, заметки склеятся. Автоматического слияния по похожим именам пока нет — оно опасно тем, что может объединить разных людей.
В релизе 08.06.2026 — нет. Импорт из CSV/Excel планируется в одном из ближайших релизов. Пока клиенты добавляются автоматически из диалогов в MAX или вручную через кабинет (по одному). Голосуйте за импорт в дорожной карте — приоритизируем по запросам клиентов.
Нет. Каждый бизнес видит только своих клиентов — это жёсткое правило, заложенное в архитектуру: все запросы к CRM фильтруются по business_id. Даже если у одного клиента есть карточки в двух бизнесах, они независимы: ваши теги, заметки и статистика принадлежат только вам.
Откройте карточку клиента и нажмите «Удалить данные клиента». Удаляются ПДн: имя, телефон, email, заметки, журнал общения. История заказов остаётся в обезличенном виде (нужна для бухгалтерии, 54‑ФЗ и возможных возвратов) — например, в аналитике останется «клиент #084, 14 заказов на 18 260 ₽», но без имени и телефона. Срок реакции по 152‑ФЗ — 30 дней; мы удаляем моментально.
Отдельно ничего. Мини‑CRM входит в единый тариф 4 990 ₽/мес вместе с ассистентом, автопостингом, каталогом, приёмом заказов, записью на услуги и ОРД с маркировкой. Никаких аддонов и скрытых платежей.
