# 168‑ФЗ — быстрый чек‑лист для публичного сайта

## Канон аббревиатур (user-facing)

| Нельзя (латиница) | Нужно (кириллица) |
|---|---|
| AI | ИИ |
| SMM | СММ |
| FAQ | Частые вопросы / ЧЗВ |
| PRO | ПРО |
| STANDARD | СТАНДАРТ |

MAX — латиницей всегда.

## Что считается user-facing

- Заголовки/тексты/кнопки/лейблы/placeholder/alt/title/aria-label.
- Микрокопи в пустых состояниях, подсказках, уведомлениях.

## Где латиница допустима

- Код, URL, домены, названия файлов, переменные, комментарии.

## Self-check команды

Примечание: для реальной проверки по проекту лучше использовать `rg` (ripgrep), но ниже — совместимо с привычным `grep`.

```bash
# 168‑ФЗ violations (user-facing и комментарии — ручная проверка по контексту)
grep -rn "\bAI\b\|\bSMM\b\|\bFAQ\b\|\bPRO\b\|\bSTANDARD\b" web/src/pages/
```

## Правило для РУМИРА

- Если слово видно пользователю — никаких латинских аббревиатур.
- Если в коде/комментариях встречается «FAQ» и т.п. — допустимо, но лучше не размножать.
